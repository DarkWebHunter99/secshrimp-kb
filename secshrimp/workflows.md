# Security Workflows - 详细参考

_按需读取，不注入系统提示。需要执行具体工作流时再 read 此文件。_

## 渗透测试工作流

1. **授权确认** — 确认有明确授权（书面/口头），确认范围和规则
2. **信息收集** — 被动侦察 → 主动扫描 → 服务识别
3. **漏洞发现** — 自动化扫描 + 手工测试，重点关注业务逻辑
4. **漏洞利用** — 最小权限原则，Get proof not shells
5. **后渗透** — 权限维持验证 → 横向移动测试 → 数据敏感性证明
6. **报告输出** — 发现 → 风险评级 → 修复建议 → PoC

## 安全运营工作流

1. **告警分诊** — 严重度评估 → 误报排除 → 关联分析
2. **威胁狩猎** — 假设驱动 → IOC 搜索 → 行为分析
3. **事件响应** — 遏制 → 取证 → 清除 → 恢复 → 复盘
4. **加固建议** — 基于实战经验给出可操作的安全加固方案

## 漏洞分析工作流

1. **CVE 追踪** — 新 CVE 发布 → 影响范围评估 → 利用难度分析
2. **PoC 验证** — 环境搭建 → 复现 → 利用条件分析
3. **防御方案** — 检测规则 → 缓解措施 → 补丁建议

## 智能体安全工作流

1. **攻击面分析** — LLM 接口 → 工具调用 → Agent 间通信
2. **Prompt Injection 测试** — 直接注入 → 间接注入 → 越狱
3. **权限滥用测试** — 工具权限边界 → 提权路径 → 数据泄露
4. **防御加固** — 输入过滤 → 输出审查 → 权限最小化
